# probemap — backlog / идеи

Короткий список на будущее (не блокирует релизы).

## Сделано (недавно)

- Палитра: вместо текста «нет на карте» — кнопка **Добавить**; стиль кнопки — нейтрально-серый.
- Добавление из палитры: раскладка по **левому краю видимой области** сверху вниз, затем колонка правее; запасной поиск спиралью.
- Клик по строке в палитре (узел на карте): **панорама к узлу** (`setCenter`), масштаб без изменений.
- Статусы по пробам, счётчик «Пробы», кружки blackbox, подсветка рамки узла, i18n — см. историю коммитов.
- **Подсказка под палитрой**: список узлов из мониторинга; ПКМ — область, узел из мониторинга или «Без метрик» (`white-space: pre-line`).
- **Терминология в UI**: «узел» в ключевых строках настроек/проекта/лейбл-карты (где имелось в виду карта, а не backend `service`).
- **Превью селектора в настройках** с учётом **фильтра активного проекта** (пары `project_filter_pairs` на `/api/config/preview-selector`, как на сервере для `/api/projects/.../services`).
- **`GET /health`** на бэкенде — ответ `{"status":"ok"}` для K8s и т.п.
- **Индикатор покрытия blackbox** в тултипе узла: строка `Blackbox: present/expected` (сколько источников дали метрики из ожидаемого списка).
- **Сбои API (не 424)**: янтарный баннер с текстом ошибки, «Повторить» / «Скрыть»; 424 по-прежнему ведёт на экран настройки источника без «красной стены».

## UX и терминология

- [ ] Пройтись ещё раз по редким экранам (модалки иконок и т.д.) на остатки слова «сервис» в смысле «узел на карте».

## Данные и модель

- [ ] Опционально: объединить типы `service` / `custom` в коде с флагом «есть метрики» — крупный рефакторинг, пока не целесообразно.

## Роли и доступ

- [ ] Роли **admin** / **editor** / **viewer**: продумать матрицу (что кому: проекты, раскладка, настройки источника, удаление/восстановление, export и т.д.) и реализовать в API + UI.

## Проекты и хранение

- [ ] Проекты **никогда не удаляются навсегда** (soft delete / корзина); **восстановление** — только у admin (или по политике из матрицы ролей).
- [ ] **Import / export** проектов (метаданные, фильтры, раскладка, при необходимости связанные `service_configs`).
- [ ] **S3** (или совместимое API) как хранилище артефактов проектов / раскладок вместо или вместе с локальными файлами в `data/`.

## Редактор карты

- [ ] **Undo / redo** последних изменений на холсте (ориентир **до ~10 шагов** в каждую сторону).

## GitOps / примеры (homelab) — вне этого репозитория

- [ ] Alloy: убрать цели, стабильно не отвечающие с ваших IP.
- [ ] Задокументировать схему 3 зон × blackbox (`zone` / `instance`) в вашем `homelab-gitops` или wiki.
